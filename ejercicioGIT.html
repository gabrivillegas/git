  1. ¿Que es un sistema de control de versiones?

    Es una herramienta que ayuda a los equipos de desarrollo de software a gestionar los cambios en el código fuente a lo largo del tiempo.

  2. ¿Cual es la diferencia entre git y github?

    Git es un sistema de control distribuido. Permite a los desarrolladores rastrear y gestionar cambios en el código a lo largo del tiempo. Y github es una plataforma de alojamiento de código basada en la web que utiliza git para el control de versiones.

  3. ¿Cómo creo una rama en git?

    Con el comando git checkout "nombre_rama"

  4. ¿Cómo clono un repositorio?

    Con el comando git clone "nombre_repositorio"

  5. ¿Cómo puedo bajar los ultimos cambios de un repositorio?

    Con el comando git push

  6. ¿Cómo puedo gener un tag?

    Con el comando git tag "nombre_tag"

  7. Cómo hago una mezcla de ramas?

    Con el comando git merge "nombre_rama"

  8. ¿Cómo puedo moverme a un commit especifico?

    Con el comando git checkout "hash_del_commit"

  9. ¿Cómo puedo borrar una rama?

    Con el comando git branch -d "nombre_rama"

  10. ¿Cómo puedo borrar un tag?

    Con el comando git tag -d "nombre_tag"

  11. ¿Cómo puedo ver los cambios que estan por enviarse al repositorio?

    Con el comando git status

  12. ¿Para que sirve el comando git remote -v?

    Te permite guardar largas direcciones URL como cortos manejadores (handles), tales como 'origin', para que no tengas que escribir las URL todo el tiempo.

    git config --global user.name "Tu Nombre" git config --global user.email
